# Coding Stress Analysis Skill

Purpose:
Analyze GitHub activity for signs of workload imbalance.

Review:

1. Commit frequency
2. Commit spikes
3. Weekend coding
4. Late-night activity
5. Number of active repositories
6. Pull request workload

Risk Indicators:

Low:
- Consistent activity
- Regular breaks

Medium:
- Large spikes of commits
- Frequent weekend work

High:
- Continuous daily activity
- Excessive commit volume
- No visible recovery periods

Output:

- Activity Summary
- Risk Assessment
- Recommendations
